Gérer les timelapses avec ansel

Mar 23, 2023 - ansel

Un timelapse durant 15 secondes contient au minimum 360 photos. Si on définit plus que 24 images par seconde, on arrive vite à un nombre important de clichés que l’on multiplie par le nombre de timelapses réalisés. Je trouve que ces nombreux clichés perturbent la vision de la collection globale. Et même si l’option de grouper les images existent, je n’y trouve pas mon compte.

Comme expliqué dans l’article de Nicolas Tissot sur la compilation, il est possible de travailler sur plusieurs versions de darktable et donc d’ansel. J’ai inversé le système en lançant à partir du même darktable plusieurs versions de ansel au niveau de la base de données. Je m’explique :

  • au niveau du disque dur, j’ai un dossier /photo qui regroupe tous les clichés et les timelapses, cela ne change pas pour la facilité des sauvegardes
  • sur une version de ansel, la collection comprend tous les clichés sauf les timelapses
  • sur une autre version de ansel, la collection ne comprend que les timelapses

Ainsi, j’ai 2 bases de données de production avec :

  • un ansel courant
  • un ansel timelapse

Au lieu de lancer ansel simplement, il faut créer un nouveau lanceur dans .local/share/applications/ par exemple ansel.desktop avec le contenu suivant :

[ Desktop Entry ]
Type = Application
Name = ansel
GenericName = ansel
Comment = logiciel de développement de photos
Exec =/opt/ansel_master/bin/darktable -- configdir /home/guillaume/.config/ansel_master/
Icon =/opt/ansel_master/share/icons/hicolor/256x256/apps/ansel.png
Terminal = false5.5. GESTION DES COLLECTIONS ET TIMELAPSE 71
Categories =
MimeType =
Keywords =
StartupWMClass =

ansel se lancera obligatoirement avec la base de données nommée après le paramètre configdir. Il suffira ensuite de créer un autre lanceur mais avec une autre base de données et un autre nom.

Et cela sans toucher à la compilation ou autres subtilités.
